Dependências e Tecnologias Detalhadas

Uma visão sobre as principais bibliotecas e ferramentas que compõem o backend da API Trace Point e como foram utilizadas.

Node.js e Express.js

O Node.js é o ambiente de execução JavaScript no servidor, permitindo a construção de aplicações backend eficientes. O Express.js é um framework minimalista e flexível para Node.js, usado para construir a estrutura da nossa API RESTful.

  • express: Framework principal para roteamento e middlewares.
    • Utilizado para: Criar a instância do servidor (express()), definir rotas (server.get(), server.post(), etc.), aplicar middlewares globais (server.use(express.json())) e middlewares de rota.
    • Importações chave: Request, Response, NextFunction, Router de express.

TypeORM e PostgreSQL

O PostgreSQL é o sistema de gerenciamento de banco de dados relacional escolhido. O TypeORM é um ORM (Object-Relational Mapper) que facilita a interação com o banco de dados de forma orientada a objetos em TypeScript.

  • typeorm: Biblioteca ORM.
    • Utilizado para: Definir entidades (@Entity, @Column, @PrimaryGeneratedColumn, etc.), criar relacionamentos (@ManyToOne, @OneToMany, @ManyToMany, @JoinColumn, @JoinTable), configurar a fonte de dados (DataSource), e realizar operações de banco (save, find, findOneBy, delete, queryRunner.query em migrações).
    • Importações chave: Decorators de entidade, DataSource, MigrationInterface, QueryRunner.
  • pg: Driver Node.js para PostgreSQL, usado pelo TypeORM para se comunicar com o banco.
    • Utilizado indiretamente pelo TypeORM.
  • reflect-metadata: Necessário para o TypeORM funcionar com decoradores e metadados de reflexão.
    • Importado globalmente (import "reflect-metadata";), geralmente no ponto de entrada da aplicação ou na configuração do DataSource.

Validação e Transformação de Dados

Para garantir a integridade e o formato correto dos dados que entram e saem da API, utilizamos as seguintes bibliotecas:

  • class-validator: Validação de objetos baseada em decoradores.
    • Utilizado para: Validar DTOs (Data Transfer Objects) aplicando decoradores como @IsNotEmpty(), @IsString(), @IsEmail(), @IsUUID(), @IsDate(), @IsEnum(), @IsOptional(), @ValidateNested(), @IsIn(), @IsNumber(), @IsPostalCode(). A função validate() é usada para executar as validações.
    • Importações chave: Decoradores de validação, validate, ValidationError.
  • class-transformer: Transformação de objetos (plain objects para instâncias de classes e vice-versa).
    • Utilizado para: Converter o corpo da requisição (plain JSON) em instâncias tipadas dos nossos DTOs (plainToInstance()). O decorator @Type() é usado para transformações de tipos aninhados ou específicos, como datas.
    • Importações chave: plainToInstance, Type.

Autenticação e Segurança

  • jsonwebtoken: Para criação e verificação de JSON Web Tokens (JWT).
    • Utilizado para: Gerar tokens de acesso e refresh (sign()) e verificar a validade e o conteúdo dos tokens recebidos (verify()).
    • Importações chave: sign, verify, JwtPayload (interface).
  • bcrypt: Para hashing de senhas.
    • Utilizado para: Gerar o hash seguro de senhas de usuários antes de armazená-las no banco (hash()) e comparar senhas fornecidas durante o login com os hashes armazenados (compare()).
    • Importações chave: hash, compare. (Você também tem @types/bcrypt nas devDependencies para tipagem).

Ambiente e Desenvolvimento

  • dotenv: Para carregar variáveis de ambiente de um arquivo .env.
    • Utilizado para: Carregar configurações sensíveis ou específicas do ambiente (dotenv.config()) para process.env.
    • Importações chave: * as dotenv.
  • typescript (devDependencies): Superset do JavaScript que adiciona tipagem estática.
    • Linguagem principal do projeto, compilada para JavaScript para produção.
  • ts-node (devDependencies): Permite executar arquivos TypeScript diretamente no Node.js sem pré-compilação explícita.
    • Utilizado para: Executar scripts da CLI do TypeORM e, possivelmente, em desenvolvimento.
  • ts-node-dev (devDependencies): Ferramenta para desenvolvimento que reinicia o servidor Node.js automaticamente quando arquivos TypeScript são alterados.
    • Utilizado no script npm run dev para agilizar o ciclo de desenvolvimento.
  • cross-env (devDependencies): Permite definir variáveis de ambiente de forma compatível entre diferentes sistemas operacionais nos scripts NPM.
    • Utilizado nos scripts NPM para configurar variáveis como TYPEORM_DATA_SOURCE para os comandos da CLI do TypeORM.
  • @types/* (devDependencies): Pacotes que fornecem definições de tipo TypeScript para bibliotecas JavaScript que não as incluem nativamente (ex: @types/express, @types/jsonwebtoken, @types/bcrypt, @types/node).
