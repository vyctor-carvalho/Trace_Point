Dependências e Tecnologias Detalhadas

Uma visão sobre as principais bibliotecas e ferramentas que compõem o backend da API Trace Point e como foram utilizadas.

Node.js e Express.js

O Node.js é o ambiente de execução JavaScript no servidor, permitindo a construção de aplicações backend eficientes. O Express.js é um framework minimalista e flexível para Node.js, usado para construir a estrutura da nossa API RESTful.

  • express: Framework principal para roteamento e middlewares.
    • Utilizado para: Criar a instância do servidor (express()), definir rotas (server.get(), server.post(), etc.), aplicar middlewares globais (server.use(express.json())) e middlewares de rota.
    • Importações chave: Request, Response, NextFunction, Router de express.

TypeORM e PostgreSQL

O PostgreSQL é o sistema de gerenciamento de banco de dados relacional escolhido. O TypeORM é um ORM (Object-Relational Mapper) que facilita a interação com o banco de dados de forma orientada a objetos em TypeScript.

  • typeorm: Biblioteca ORM.
    • Utilizado para: Definir entidades (@Entity, @Column, @PrimaryGeneratedColumn, etc.), criar relacionamentos (@ManyToOne, @OneToMany, @ManyToMany, @JoinColumn, @JoinTable), configurar a fonte de dados (DataSource), e realizar operações de banco (AppDataSource.manager.save(), .find(), .findOneBy(), .delete(), ou via repositórios; queryRunner.query() em migrações).
    • Importações chave: Decorators de entidade, DataSource, MigrationInterface, QueryRunner, Column, Entity, JoinColumn, JoinTable, ManyToMany, OneToMany, OneToOne, PrimaryGeneratedColumn, PrimaryColumn.
  • pg: Driver Node.js para PostgreSQL, usado pelo TypeORM para se comunicar com o banco.
    • Utilizado indiretamente pelo TypeORM; listado como dependência no package.json.
  • reflect-metadata: Necessário para o TypeORM funcionar com decoradores e metadados de reflexão do TypeScript.
    • Importado globalmente uma vez (import "reflect-metadata";), geralmente no ponto de entrada da aplicação (ex: server.ts ou index.ts) antes de qualquer uso do TypeORM.

Validação e Transformação de Dados

Para garantir a integridade e o formato correto dos dados que entram e saem da API, utilizamos as seguintes bibliotecas:

  • class-validator: Validação de objetos baseada em decoradores.
    • Utilizado para: Validar DTOs (Data Transfer Objects) aplicando decoradores como @IsNotEmpty(), @IsString(), @IsEmail(), @IsUUID(), @IsDate(), @IsEnum(), @IsOptional(), @ValidateNested(), @IsIn(), @IsNumber(), @IsPostalCode(). A função validate() é usada externamente (no seu utilitário validateRequestBody) para executar as validações definidas nos DTOs.
    • Importações chave (nos DTOs): Diversos decoradores de validação. validate e ValidationError são usados no seu utilitário.
  • class-transformer: Transformação de objetos (plain objects para instâncias de classes e vice-versa).
    • Utilizado para: Converter o corpo da requisição (plain JSON) em instâncias tipadas dos nossos DTOs (plainToInstance()). O decorator @Type() é usado nos DTOs para transformações de tipos aninhados ou específicos (ex: de string para Date).
    • Importações chave: plainToInstance, Type (decorator).

Autenticação e Segurança

  • jsonwebtoken: Para criação e verificação de JSON Web Tokens (JWT).
    • Utilizado para: Gerar tokens de acesso e refresh (sign()) e verificar a validade e o conteúdo dos tokens recebidos (verify()) no TokenManager.
    • Importações chave: sign, verify, JwtPayload (interface).
  • bcrypt: Para hashing de senhas.
    • Utilizado para: Gerar o hash seguro de senhas de usuários antes de armazená-las no banco (hash()) e comparar senhas fornecidas durante o login com os hashes armazenados (compare()).
    • Importações chave: hash, compare. (@types/bcrypt para tipagem).

Ambiente, Desenvolvimento e Utilitários

  • dotenv: Para carregar variáveis de ambiente de um arquivo .env.
    • Utilizado para: Carregar configurações sensíveis ou específicas do ambiente (dotenv.config()) para process.env, tipicamente no início da aplicação (EsportEnv.ts).
    • Importações chave: * as dotenv ou import { config } from "dotenv".
  • typescript (devDependencies): Superset do JavaScript que adiciona tipagem estática.
    • Linguagem principal do projeto, compilada para JavaScript para produção (via comando tsc).
  • ts-node (devDependencies): Permite executar arquivos TypeScript diretamente no Node.js.
    • Utilizado para: Executar scripts da CLI do TypeORM e em desenvolvimento.
  • ts-node-dev (devDependencies): Ferramenta para desenvolvimento que reinicia o servidor Node.js automaticamente quando arquivos TypeScript são alterados.
    • Utilizado no script npm run dev.
  • cross-env (devDependencies): Permite definir variáveis de ambiente de forma compatível entre diferentes sistemas operacionais nos scripts NPM.
    • Utilizado nos scripts NPM para configurar variáveis como TYPEORM_DATA_SOURCE ou TYPEORM_MIGRATIONS para os comandos da CLI do TypeORM.
  • @types/* (devDependencies): Pacotes que fornecem definições de tipo TypeScript para bibliotecas JavaScript (ex: @types/express, @types/jsonwebtoken, @types/bcrypt, @types/node).
